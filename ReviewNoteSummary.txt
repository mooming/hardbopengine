CRITICAL BUGS (MUST FIX IMMEDIATELY)
================================================================================

1. MEMORY CORRUPTION: VirtualFree uses wrong deallocation on Linux/macOS
   - Files: Engine/OSAL/LinuxMemory.cpp:27, Engine/OSAL/OSXMemory.cpp:29
   - Issue: Memory allocated with mmap() is freed with free() instead of munmap()
   - Impact: Undefined behavior, potential heap corruption, crashes
   - Fix: Replace free(address) with munmap(address, 0)

2. INVERTED ERROR CHECK: VirtualFree crashes on SUCCESS (Windows)
   - Files: Engine/OSAL/OSMemory.cpp:112, Engine/OSAL/WindowsMemory.cpp:37
   - Issue: if (unlikely(result)) should be if (unlikely(!result))
   - Impact: Engine crashes whenever VirtualFree succeeds
   - Fix: Add negation operator to condition

3. COMPILATION ERROR: Undefined variable 'size' in VirtualFree
   - File: Engine/OSAL/OSMemory.cpp:28
   - Issue: Parameter unnamed but body references 'size'
   - Impact: Code should not compile on Linux
   - Fix: Name the parameter or use 0 (munmap ignores size for full unmap)

4. UNIMPLEMENTED FUNCTIONS: String::Replace/ReplaceAll assert false
   - File: Engine/String/String.cpp:617-620, 650-653
   - Issue: Functions marked TODO but not stubbed safely
   - Impact: Runtime crash in debug builds if called
   - Fix: Either implement or throw/return error gracefully

5. FORMAT STRING BUG: "%ud" is invalid
   - File: Engine/String/String.cpp:86
   - Issue: %ud is not a valid printf format specifier
   - Impact: Produces incorrect output (e.g., "5d" instead of "5")
   - Fix: Change to %u or %hu depending on type
