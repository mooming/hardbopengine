================================================================================
HARDBOP ENGINE CODE REVIEW - Config Module
================================================================================

OVERVIEW: Configuration system with key-value parameters and file-based settings.

FILES:
  - BuildConfig.h     - Build-time configuration flags
  - ConfigParam.h    - Thread-safe config parameter template
  - ConfigSystem.h   - Global config registry
  - ConfigFile.h     - File-based configuration
  - EngineConfig.h  - Engine-specific settings

STRENGTHS:
  + Template-based ConfigParam with optional atomic support
  + Conditional compilation for parameter descriptions (ENGINE_PARAM_DESC_ENABLED)
  + Thread ID tracking in debug builds for thread safety validation

ISSUES:
  - LOW: MAX_NUM_TASK_STREAMS macro has trailing semicolon (BuildConfig.h:16)
  - LOW: __DEBUG__ macro logic has unnecessary #undef and mismatched comments (BuildConfig.h:19-22)
  - Constructor uses conditional initialization - could use delegating constructors
  - std::mutex in ConfigParam may cause contention in high-frequency access

STYLE CONCERNS:
  - Mix of camelCase and PascalCase in internal types (TValue vs template params)

================================================================================
