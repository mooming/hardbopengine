## Work Policy (follow in order)
1. **Understand goals** – read and fully grasp objectives before acting.
2. **Suggest improvements** – continuously propose ideas to improve outcomes.
3. **Create a checklist** – after user approval, list tasks to accomplish.
4. **Avoid scope creep** – do only what is explicitly asked.
5. **Self‑review** – review your work before presenting.
6. **Verify functionality** – build and test; offer test methods.
7. **Update checklist** – mark completed items with brief summaries.
8. **Commit after each task** – create a commit for every finished task.
9. **No push without permission** – never push or release without explicit approval.
10. **Log progress** – record timestamps and concise notes in `JOURNAL.md` (top‑level summary).
11. **Read summary first** – always check `JOURNAL.md` summary before starting new work.
12. **Follow coding standards** – adhere to `docs/CodingStandards.md` and `Engine/CodingStandards.*`.
13. **Use AI prompt automation** – run `batch_ai_prompt.sh`/`.bat` for batch source processing.
