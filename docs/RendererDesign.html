1. Architecture Overview

The renderer follows a strategy + factory pattern: a single abstract interface (IRenderer) with concrete backends selected at runtime via platform-conditional factory logic. The renderer is not owned by the core Engine class — application code creates and manages it directly, reflecting a deliberate decoupling between engine subsystems and the graphics layer.

9. Metal Backend Deep-Dive

Resource Handles (Opaque)

All GPU resources are stored as intptr_t to avoid leaking Objective-C types into the C++ header:

class MetalRenderer : public IRenderer {
    intptr_t device;          // id<MTLDevice>
    intptr_t commandQueue;    // id<MTLCommandQueue>
    intptr_t pipelineState;   // id<MTLRenderPipelineState>
    intptr_t vertexBuffer;    // id<MTLBuffer>
    intptr_t metalLayer;      // CAMetalLayer*
};

Initialization Sequence

Figure 8: Metal initialization sequence — 6 steps from device creation to initial vertex buffer.

EndFrame GPU Work

MetalRenderer::EndFrame()
  1. [metalLayer nextDrawable]                    // get backbuffer texture
  2. [commandQueue commandBuffer]                 // create command buffer
  3. MTLRenderPassDescriptor:
       colorAttachment[0].texture = drawable.texture
       loadAction = Clear (.storeAction = Store)
       clearColor = (0.1, 0.1, 0.15, 1.0)
  4. [commandBuffer renderCommandEncoderWithDescriptor]
       setRenderPipelineState
       setVertexBuffer (offset:0, index:0)
       drawPrimitives (triangle, 0, 6)
       endEncoding
  5. [commandBuffer presentDrawable] + commit
  6. [commandBuffer waitUntilCompleted]           // ← CPU blocks

Rotation Shader (compiled at runtime)

// Inline Metal shader — vertex + fragment
vertex VertexOut vertexShader(uint vertexID [[vertex_id]],
                     constant float *vertices [[buffer(0)]]) {
    VertexOut out;
    float2 pos = float2(vertices[vertexID*6], vertices[vertexID*6+1]);
    out.position = float4(pos, 0.0, 1.0);
    out.color = float4(vertices[vertexID*6+2], vertices[vertexID*6+3],
                       vertices[vertexID*6+4], vertices[vertexID*6+5]);
    return out;
}
fragment float4 fragmentShader(VertexOut in [[stage_in]]) {
    return in.color;
}

PrincipleHow It's Applied
Composition over inheritanceFuture subsystems compose IRenderer rather than deriving from it
noexcept correctnessAll renderer API methods are noexcept
RAIIstd::unique_ptr<IRenderer> for lifecycle, CFBridging for ObjC ownership
